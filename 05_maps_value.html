Colour maps by continuous value#

This notebook shows an example of creating a map in matplotlib where each region is drawn in a different colour to match a given value.

In this example we will plot the travel time from each LSOA to a chosen stroke unit.

Plain English summary#

A nice way to see variation around the country is to plot a map and give each small area a different colour depending on some value. This value could be for example the travel time to the nearest hospital, or the average disability score after a stroke, or the average age of people in that area.

These types of data are ‚Äúcontinuous‚Äù - rather than being whole numbers like 87, 88, or 89, the values could be anywhere in between like 87.123456789. This means that even a slight change in value between two different areas will mean that they are given slightly different colours on the map.

Usually we select the colours from a standard colour scheme. These colour schemes make it so that two very similar values will have very similar colours, and two very different values will have different colours. We usually plot either the lowest values in the darkest colours and go gradually lighter in shade up to the highest values, or the opposite (lowest values are pale, highest values are dark).

In the example in this notebook we will plot the colours varying with travel time to the nearest stroke unit.

Aims#

Plot the variation of travel time to the nearest stroke unit around England and Wales.

Method#

  • Load a list of the LSOA around England and Wales

  • Create the travel times to the nearest stroke unit

  • Plot the varying travel time on a map

Make map#

Convert the LSOA geometry to British National Grid coordinate system to match the stroke units:

gdf_lsoa = gdf_lsoa.to_crs('EPSG:27700')

Find shared colour limits:

vmin = gdf_lsoa[['unit_travel_time', 'unit_travel_time_mt']].to_numpy().min()
vmax = gdf_lsoa[['unit_travel_time', 'unit_travel_time_mt']].to_numpy().max()

print(vmin, vmax)

5.8 146.5

# Give the colourbar its own axis otherwise it'll squeeze into
# an existing axis and make the maps look different sizes.
# Use the gridspec keywords to make the width much less for
# the colourbar axis.
fig, axs = plt.subplots(
    1, 3, figsize=(12, 8),
    gridspec_kw={'width_ratios': [1, 1, 0.05]}
)

ax = axs[0]
ax1 = axs[1]
ax_cbar = axs[2]

# ----- Left-hand map -----
# Catchment areas:
gdf_lsoa.plot(
    ax=ax,              # Set which axes to use for plot
    antialiased=False,  # Avoids artefact boundary lines
    column='unit_travel_time',  # set colours using this column
    cmap='inferno',
    edgecolor='face',
    vmin=vmin,
    vmax=vmax
    )
# Stroke units:
ax.scatter(
    gdf_units.loc[mask_acute, 'geometry'].x,
    gdf_units.loc[mask_acute, 'geometry'].y,
    marker='o',
    facecolor='w',
    edgecolor='k',
    label='Stroke unit',
    s=20,  # size
    )

# ----- Right-hand map -----
# Catchment areas:
gdf_lsoa.plot(
    ax=ax1,              # Set which axes to use for plot
    antialiased=False,  # Avoids artefact boundary lines
    column='unit_travel_time_mt',  # set colours using this column
    cmap='inferno',
    edgecolor='face',
    legend=True,  # to draw colourbar
    legend_kwds={'shrink': 0.5, 'label': 'Travel time (minutes)', 'cax': ax_cbar},
    vmin=vmin,
    vmax=vmax
    )
# Stroke units:
ax1.scatter(
    gdf_units.loc[mask_mt, 'geometry'].x,
    gdf_units.loc[mask_mt, 'geometry'].y,
    marker='o',
    facecolor='w',
    edgecolor='k',
    label='Stroke unit',
    s=20,  # size
    )

# Labels
ax.set_title('Closest acute stroke unit')
ax1.set_title('Closest MT unit')
# Remove the axis border, ticks, labels...
ax.axis('off')
ax1.axis('off')

# Add legend:
ax1.legend(bbox_to_anchor=[1.0, 0.0], loc='upper right')

plt.show()
